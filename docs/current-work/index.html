Repository-safety redesign of the installer footprint

complete Verified from source
Objective
Make `vibekb install .` safe in fresh, active, and mature repositories — consolidate everything VibeKB owns under `.vibekb/`, integrate with shared files only via namespaced adapters or a marked managed block, and add migrate/uninstall/doctor + a per-install manifest.

Data impact
Changes the installed footprint in TARGET repositories (everything now under `.vibekb/`; shared files touched only via a managed block). VibeKB's own self-hosted layout is unchanged. Introduces `.vibekb/install.json` (replaces legacy `.vibekb/.installer.json`).
Risks
Two supported PHP layouts (self-hosted root vs consolidated `.vibekb/runtime/`) resolved by `guide/lib/workspace.php`; self-hosted behavior kept byte-identical and re-verified with `check`. Migration of legacy installs is content/hash-gated; ambiguous files are preserved, not deleted.
