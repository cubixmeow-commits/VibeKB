Constraints

  • All content file access is confined to the content root

    Verified from source

    The loader refuses any path that escapes the configured `.vibekb/` root, and record ids are constrained to a safe character set, so a crafted request can never read arbitrary files.

    Load the living model Render the dynamic guide

  • Runs on PHP 8.2 shared hosting with no build step

    Verified from source

    No framework, SPA, bundler, database, external/AI API, or JavaScript build; the guide must run by copying files and work in a subfolder without JavaScript.

Warnings

  • /docs is generated output — never hand-edit it

    medium Verified from source

  • The model drifts from the code unless an agent reconciles it

    high Verified from source

    VibeKB detects that files changed but cannot interpret what a change means; if an agent edits code without updating the affected records, the model silently goes stale.

    high Verified from source
